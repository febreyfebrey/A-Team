# Report Schedule

## 規則內容

### 週報排程（Weekly Report Schedule）

你必須在每週一產生週報。週報涵蓋上一週（週一至週日）的資料。

產生時限：每週一上午 10:00 前完成。

### 月報排程（Monthly Report Schedule）

你必須在每月第一個週一產生月報。月報涵蓋上一個完整月份的資料。

產生時限：當月第一個週一上午 10:00 前完成。

### 月報與週報的關係

當月第一個週一同時觸發週報與月報。此時你必須產生兩份報告：

1. **週報**：涵蓋上一週的資料
2. **月報**：涵蓋上一個完整月份的資料，必須包含週報的全部五個章節（執行摘要、優先級排序、停滯警示、開發進度、工單建立統計），另加趨勢分析（Trend Analysis）與月度比較（Month-over-Month Comparison）

月報不取代週報。兩份報告必須同時產生。

### 時限規範

- 上午 10:00 指的是當地時間（Local Time）
- 超過 10:00 才完成的報告視為遲交
- 報告必須在資料收集完成後立即產生，不得延遲

### 報告日期標記

- 週報的報告日期（Report Date）標記為該週一的日期
- 月報的報告日期標記為該月第一個週一的日期

## 違規判定（Violation Determination）

- 週一未產生週報 → 違規
- 每月第一個週一未產生月報 → 違規
- 報告在上午 10:00 後才完成 → 違規
- 月報缺少趨勢分析或月度比較章節 → 違規
- 每月第一個週一僅產生月報而未同時產生週報 → 違規

### 違規情境

2025 年 4 月 7 日為該月第一個週一。團隊僅產生了一份月報，認為月報已包含週報內容因此無需另外產生週報。此做法違規，因為月報與週報為獨立的兩份報告，必須同時產生。即使月報包含週報的全部章節，週報仍須作為獨立文件產生。
